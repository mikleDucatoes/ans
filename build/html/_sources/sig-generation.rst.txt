Генерация удостоверяющей подписи
================================

Для обращения по API iWine необходимо иметь в качестве обязательного параметра подписи запросов (Sig), который использования на стороне встроенного web-сервера для распознавания «свой-чужой».
Sig формируется по правилу:
 #. Необходимо сформировать первую часть подписи p1Sig, используя md5(текущая дата в формате POSIX/UNIX-time - это большое целое число).
 #. Далее берем 6-ти символьный код p2Sig, который располагается на обложке паспорта устройства. В данных символах зашифрована серия и номер устройства, что обеспечивает уникальность устройства.
 #. Конкатенируем «p1Sig» и «p2Sig» (дописываем в конец к «p1Sig» код с паспорта устройства «p2Sig»).
Вы получили подпись Sig для обращения к API-интерфейсу iWine. Sig предназначена для распознавания приходящих на HTTP-сервер iWine запросов:

 * «своих» - запросы с корректной Sig, по которым будет высылаться ответ;
 * «чужих» - запросы с неверной Sig, по которым не происходит отправки ответа.
